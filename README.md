# 🚀 CI/CD Automated Deployment Project

## 📌 Project Overview
This project demonstrates how to eliminate manual deployment by implementing a fully automated CI/CD pipeline.  
Every code update is automatically detected, built, tested, and deployed to the live application without human intervention.

---

## ⚙️ Architecture

Code Commit → GitHub → Jenkins → Build & Test → Deployment → Live Application

---

## 🎯 Objectives

- Detect code changes automatically  
- Deploy updates without manual effort  
- Ensure consistency and reliability  
- Implement real-world DevOps practices  

---

## 🛠️ Tools & Technologies

- GitHub (Version Control)
- Jenkins (CI/CD Automation)
- Nginx (Web Server)
- Linux Server (Deployment Environment)

---

## 🚀 What I Did

- Connected GitHub repository with Jenkins using webhook  
- Created a Jenkins pipeline for automation  
- Configured Nginx server for hosting  
- Automated code deployment process  
- Tested pipeline with multiple commits  

---

## ⚡ How It Works

1. Developer pushes code to GitHub  
2. GitHub webhook triggers Jenkins automatically  
3. Jenkins pulls latest code using `git pull`  
4. Build process executes  
5. Application is deployed to server  
6. Live application updates instantly  

---

## ✅ Key Features

- Fully automated deployment  
- No manual intervention  
- Faster delivery of updates  
- Reliable and consistent deployment
